## 1. Краткая суть проекта

Первокурсник или любой студент пишет в чат-бот любой вопрос про УдГУ — и мгновенно получает точный, понятный и полезный ответ. 
Бот знает всё, что студенту может понадобиться: от расписания и оформления справок до расположения корпусов и работы ИИАС.
И всё это работает на обученной языковой нейросетевой модели, обученной на контексте УдГУ.

## 2. Проблему, которую решает проект

Когда первокурсники приходят в УдГУ, они сталкиваются с большим количеством сложностей:
-	не понимают, куда обращаться за документами,
-	не знают, где находятся корпуса,
-	путаются в ИИАС, distEdu и сайте УдГУ,
-	стесняются спрашивать у кураторов,
-	долго ищут нужную информацию.

Из-за этого первокурсники испытывают стресс и студентам трудно ориентироваться в университетской среде.

## 3. Цель проекта

Создать систему из обученной на контексте УдГУ языковой модели, базы данных и чат-бота, которая сможет исправно работать и в большинстве случаев (95%) правильно отвечать на вопросы пользователей.

## 4. Ключевые функции

Бот знает и умеет:

Навигация по университету
-	расположение корпусов и аудиторий,
-	краткое описание каждого корпуса,
-	как дойти до нужного места.

Учебные вопросы
-	расписание занятий,
-	информация о курсах,
-	пересдачи, сессия, сроки.

Документы и справки
-	как оформить академический отпуск,
-	как получить справки,
-	какие заявления существуют и куда их сдавать.

Порталы
-	официальный сайт УдГУ,
-	ИИАС,
-	distEdu,
-	инструкции по использованию порталов.

Контакты
-	деканаты, отделы, службы, канцелярии, приёмные, библиотеки.

Простота обновления
Администратор может менять в базе данных:
-	ссылки,
-	документы,
-	текстовые материалы
— без изменения логики и без переобучения модели.

## 5. Сценарии использования

1. Первый заход пользователя чат сопровождается кнопкой "/start"
2. Если писать боту без нажатия на кнопку "/start", то бот отвечать не будет
3. После нажания на кнопку "/start" бот отправляет приветственное сообщение
4. Дальше пользователь может написать боту любой текстовый запрос
5. Если запрос не относится по теме к УдГУ, то бот выкидывает сообщение, что запрос не по теме и он не знает ответа
6. Если запрос по теме, то бот выдает текстовое сообщение с ответом и кнопками, в которых пишутся темы по вопросу уточняющие
7. Если пользователь написал новое сообщение, то идем на пункт 4.
8. Если пользователь нажала на кнопку с уточняющей темой, то идем на пункт 6.

## 6. Технологическая часть 

-	Бот работает в VK.
-	Используется лёгкая офлайн-модель или подключаемая LLM для понимания вопросов.
-	Запрос классифицируется по темам.
-	Ответы берутся из базы данных (PostgreSQL)


## 7. Примерный объём работ (не окончено)

Работа над проектом распределена на 5 спринтов:
1. Первый рабочий прототип приложения
2. Разработка языквой модели и работа с ней
3. сбор информации об удгу
4. добавление остальных фич
5. тестировка и полировка
