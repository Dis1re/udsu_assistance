# Пайплайн работы приложения:
Пользователь спрашивает бота в вк -> бот берет сообщение и кидает его в бэкенд -> бэкенд кидает сообщение в нейронку обученную -> нейронка понимает контекст сообщения и выдает обратно в бэкенд теги для вывода ответа и сопроводительный текст -> бэкенд по тегам вытягивает информацию из базы данных, формирует ответ для бота и кадает сообщение боту -> бот кидает это всё пользователю -> пользователь получает верное сообщение и счастливый идет дальше

## структура json_request для post запроса от бота к серверу
{
    "user_id": "user_id"
    "type": "text" || "tag", // text - текст вопроса пользователя, tag - если пользак нажал на кнопку с тэгом
    "text": "text" || "tag_text" // соответствующий текст вопроса пользователя или строгий tag вопроса,              
                                 // по которому потом он обратится бд        
}

## структура json_response для post ответа от сервера боту
{
    "result": "ok" | "error",
    "data": {
        "text": "text from back, db, neuro"   // базовый текст, который видит пользователь
        "buttons": [ // кнопки, которые бот получает по контексту от сервера
            {
                "action": {
                    "type": "callback",
                    "payload": "tag", // это информация, которая обработается ботом внутри, 
                                      //пользак её не видит, то есть здесь должен быть полный tag как в бд
                    "label": "tag_text" // текст из тэга, каторый увидит пользователь
                },
                "color": "primary"
            },
            ...
        ]
    },
    "error"?: "error text" // только когда ошибка
}

# Вк бот
## Язык: Python
## Требования к боту:
1) отдавать сообщения от пользователя в бэкенд.
2) принимать сообщения от бэкенда и отправлять их обратно пользователю.
## подробно
1. бот запускается
2. бот принмает сообщение от пользователя через апи
3. бот форматирует это сообщение под формат jsonс типом "text" (чуть ниже json_request)
4. бот получает из файла config.py в котором написаны url сервера
5. бот по полученным данным сервера и сформированному json кидает пост запрос в сервер
6. бот ждет ответа от сервера в формате json (чуть ниже json_response)
7. бот расформатирывает сообщение из json на текст и кнопки
8. бот составляет сообщение, оно должно составлять из себя текст и кнопки (если они есть)
9. бот ждет нового сообщение от пользователя или нажатия на кнопку
10. бот принимает от пользователя новое сообщение -> идем на пункт 2, иначе 11.
11. бот принимает от пользователя нажатие на кнопку
12. бот обрабатывает нажатие на кнопку и отправляет в бэк соответствующий запрос с типом "tag", дальше идем на пункт 6.

# бэкенд
## язык: TypeScript
## Требования к бэкенду:
1)	принимать сообщение от вк бота и отдавать новые сообещния обратно
2)	отдавать запрос в нейронку и получать ответ от нейронки
3)	должен уметь отдавать запрос на получение данных из бд и получать его
4)	Формировать конечное сообщение, основанное на тексте полученного от нейронки и информации полученной от запроса из бд

## подробно
1. бэк получает пост запрос от бота в форме json_request
2. бэк различает тип сообщения по обычному тексту и тэгу
3. если сообщение текст, то он его кидает в нейронку
4. от нейронки получает ответ в форме {текст: текст, тэги: тэги} и проходится по тэгам и в бд кидает
5. получает от бд текст тэгов и формирует ответ в форме json_responce, где в текст кидается текст от нейронки
   а в button кидает текст тэгов и сами тэги
6. и сформированный ответ кидает в обратку боту, дальше идем на пункт 1.
7. если сообщение типа тэг, то он сразу кидает тэги в бд
8. получает ответ по тегам, и формирует ответ в форме json_responce, где в текст кидает заготовленный ответ 
   по типу "выберите вариант", а в button полученные тэги и тексты тэгов
9. кидаем сформированный ответ боту и идем на пункт 1.
